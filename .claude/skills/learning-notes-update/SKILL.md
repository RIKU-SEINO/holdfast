# learning-notes-update — 学習ノートの更新

holdfast は「作りながら学ぶ」リポ。`docs/learning-notes/phase-N.md` に各 Phase の
学習メモがあり、実装で得た知見を蓄積する。この skill はその各節を**今セッションの
実作業から**追記更新する。

## 対象の節

| 節 | 何を書くか |
|---|---|
| ハマりメモ | 実装中に詰まったこと（症状 → 原因 → 直し方） |
| 設計メモ | なぜそう設計したか（選んだ理由・捨てた代替案） |
| ミスログ | やらかし表（# / やらかし / 原因 / 覚えること） |
| 実験メモ | 計測結果の表（あれば実データで埋める） |
| クイズ②（実装後） | 実装で答えられるようになった問いの回答 |

## 原則

- **実作業に基づく。** 今セッションで実際に起きたこと（踏んだエラー、下した設計判断、
  犯したミス、計測した数値）だけを書く。一般論・教科書的な知識を捏造して埋めない。
  該当することが何も起きていない節は**空のまま残す**。
- **追記であって上書きでない。** 既存の記述は消さない・改変しない。新しい項目を足すだけ。
  書く前に必ず該当節を Read し、**既出の項目と重複しないか**確認する。
- **中途半端を正直に書く。** 作業が途中・未解決のまま終わるセッションもある。その場合は
  でっち上げて完成させず、`（未解決）`『途中：〜まで確認、〜が残り』のように**状態を明示**する。
  DoD を満たしていないのに満たしたかのように書かない。実験表は未計測なら空セルのままでよい。
- **簡潔に。** 1 項目 1〜3 行。症状・原因・対処を端的に。後で自分が読み返して思い出せる粒度。
- **学習者の言葉を尊重する。** ユーザー自身の理解の記録なので、Claude の解説をそのまま
  貼るのではなく、要点を箇条書きで残す。専門用語の対応（happens-before 等）は明記してよい。

## 手順

1. **対象 Phase を特定する。** 引数や文脈、CLAUDE.md の「現在地」から `phase-N.md` を決める
   （不明なら聞く）。ファイルを Read する。

2. **既存の学習メモ各節を読む。** 何がすでに書かれているか把握し、重複を避ける。

3. **今セッションの出来事を棚卸しする。** この会話で実際に起きた
   「詰まり・設計判断・ミス・計測値・新たに answer できるクイズ②」を洗い出す。
   何も該当しない節は触らない。

4. **各節に追記する。** Edit で該当節に項目を足す。既存項目は残す。中途半端なものは
   状態（未解決・途中）を明示。

5. **更新内容を要約報告する。** どの節に何を足したか、未解決として残したものは何かを伝える。

## やらないこと

- 既存の記述を上書き・削除しない。
- 起きていないことを書かない（捏造しない）。該当なしの節は空のまま。
- 途中の作業を完成扱いにしない。未解決は未解決と書く。
